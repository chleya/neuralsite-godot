## 一、项目愿景

> **"打造一款沉浸式的道路桥梁施工4D可视化系统，让项目经理和施工团队能够直观地看到'时间'维度上的施工进度，预测延误，优化调度。"**

## 三、行业对标

### 3.1 4D施工模拟行业标准

参考: Autodesk Navisworks, Bentley Synchro, Fuzor

**核心功能要求**:
1. ✅ 3D可视化
2. ✅ 时间轴模拟
3. ✅ 碰撞检测
4. ⚠️ 实时数据同步
5. ⚠️ 多用户协作
6. ⚠️ 进度报告自动生成

### 3.2 差距分析

| 行业标准 | 项目现状 | 差距 |
|---------|---------|------|
| 真实工地场景 | 有地形/道路/桥梁 | 中等 - 缺少施工机械细节 |
| 进度模拟 | 有时间轴 | 较好 |
| 碰撞检测 | 有AABB简化版 | 中等 - 需精确化 |
| 实时同步 | Mock完成 | 低 - 需真实API |
| 施工动画 | 基础动画 | 低 - 需丰富 |
| 报表导出 | 无 | 低 - 需开发 |

## 五、详细实施计划 (M1: MVP)

### Phase 1.1: 代码清理 (2天)

**目标**: 消除冗余代码，降低维护成本

```
Day 1:
  ├── 删除 ConstructionTimeline.gd (保留 TimelineManager)
  ├── 删除 PrecisionCollisionSystem.gd (保留 CollisionDetector)
  └── 统一实体基类 (保留 ConstructionEntity)

Day 2:
  ├── 更新 Main.gd 中的引用
  ├── 更新 README.md
  └── 提交: "refactor: remove redundant timeline and collision systems"
```

### Phase 1.2: 数据导入完善 (3天)

**目标**: 让 GeoJSON 导入真正可用

```
Day 3:
  ├── 扩展 GodotImporter 支持更多字段
  ├── 添加验证逻辑
  └── 添加错误处理

Day 4:
  ├── 扩展 GodotExporter 导出完整数据
  ├── 添加导出格式选项 (GeoJSON, CSV)
  └── 添加导出预览

Day 5:
  ├── 测试导入导出流程
  ├── 修复边界问题
  └── 提交: "feat: complete GeoJSON import/export pipeline"
```

### Phase 1.3: API对接 (3天)

**目标**: 连接真实后端

```
Day 6:
  ├── 实现 APIClient 的所有接口

Day 7:
  ├── 实现 WebSocket 实时更新
  ├── 添加断线重连

Day 8:
  ├── 创建后端模拟服务器 (用于测试)
```
